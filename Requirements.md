# Kirana Pro | किराना प्रो - Technical Requirements Specification

## 1. Executive Summary
This document outlines the technical requirements, software architecture, and product roadmap for **Kirana Pro**, a modern Inventory Management and POS system for Indian retail. The goal is to provide a high-performance, resilient, and user-friendly platform that operates seamlessly across mobile and desktop.

---

## 2. Recommended Technology Stack
To ensure speed, offline resilience, and rapid development, the following "latest and best" stack is recommended:

### 2.1 Frontend
- **Framework**: **Next.js 14+ (App Router)** for optimized performance and SEO (if a web version is needed).
- **Styling**: **Tailwind CSS** for rapid UI development and utility-first styling.
- **State Management**: **Zustand** for lightweight, performant global state (Cart, User, Stock).
- **Component Library**: Custom implementation based on **Bharat Retail Core** (using Radix UI or Headless UI for accessibility).
- **Bilingual Support**: **i18next** with specialized Hindi/English JSON dictionaries.

### 2.2 Mobile Development
- **Cross-Platform**: **React Native** (using Expo) for true native performance on Android and iOS.
- **Offline Storage**: **WatermelonDB** or **SQLite** for high-performance local data persistence with synchronization capabilities.

### 2.3 Backend & Infrastructure
- **Serverless**: **Supabase** (PostgreSQL) for Authentication, Real-time Database, and Edge Functions.
- **API**: **PostgREST** (auto-generated by Supabase) or **GraphQL** for flexible data fetching.
- **Deployment**: **Vercel** (Web) and **Expo EAS** (Mobile).

---

## 3. Functional Requirements

### 3.1 Authentication & User Management
- **Phone Number Login (OTP)**: Primary login method for Indian shopkeepers (via Firebase Auth or Supabase).
- **Role-Based Access Control (RBAC)**:
  - **Owner**: Full access to profits, reports, and staff management.
  - **Cashier**: Access to Billing and limited Inventory view.
  - **Staff**: View stock and add/receive inventory only.

### 3.2 Core Modules
#### A. Fast Billing (POS)
- **Barcode Scanning**: Integration with device camera for instant product lookup.
- **Cart Logic**: Handle quantity increments, manual price overrides (if allowed), and discount application.
- **Payment Integration**: Generate dynamic UPI QR codes for instant payments.

#### B. Inventory Management
- **Dynamic Calculation**: Real-time margin calculation: `(Selling - Purchase) / Purchase * 100`.
- **Low Stock Engine**: Background job to flag items falling below the `Min Alert` threshold.
- **Batch & Expiry**: Support for multiple batches per product with FEFO (First Expired, First Out) logic.

#### C. Financial Engine
- **Ledger System**: Double-entry bookkeeping for Cash Balance tracking.
- **Expense Logging**: Daily tracking of Rent, Salaries, and Electricity.

---

## 4. Non-Functional Requirements

### 4.1 Performance
- **TBT (Total Blocking Time)**: < 100ms on billing screens to prevent lag during rush hours.
- **Local-First**: All billing operations must work without an internet connection, syncing when online.

### 4.2 Security
- **Data Privacy**: Shop data must be isolated and encrypted at rest.
- **Audit Logs**: Every sale and inventory adjustment must be timestamped and attributed to a user ID.

### 4.3 Accessibility
- **Bilingualism**: 100% coverage of Hindi and English.
- **Haptic Feedback**: Vibration on successful barcode scan and sale completion.

---

## 5. Deployment & CI/CD
- **Automated Testing**: Jest for unit tests; Playwright for E2E flows (Billing/Checkout).
- **Monitoring**: Sentry for error tracking and PostHog for product analytics.

---

## 6. Implementation Roadmap

### Phase 1: MVP (Weeks 1-4)
- User Auth (OTP).
- Basic Product CRUD.
- Fast Billing with Cash/UPI buttons.
- Simple Dashboard (Sales today).

### Phase 2: Optimization (Weeks 5-8)
- Barcode scanning integration.
- Expense management module.
- Offline-sync engine implementation.
- Basic PDF report generation.

### Phase 3: Advanced Features (Weeks 9-12)
- Supplier Ledger.
- Customer Udhaar (Credit) tracking.
- Multi-staff support.
- Advanced Analytics (Profit/Loss trends).
